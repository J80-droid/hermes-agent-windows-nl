# Security Procedures

## Standard Operating Procedures (SOP)

### 1. Vulnerability Assessment

1. Scope definiëren (systemen, netwerksegmenten, tijdlijn)
2. Tool selectie (gebaseerd op doel)
3. Scan uitvoeren in geautoriseerde tijdslot
4. Resultaten analyseren
5. Risico classificeren (CVSS)
6. Rapport opstellen met mitigatie-advies
7. J. presenteren

### 2. Penetration Testing

1. Scope en rules of engagement definiëren
2. Test-omgeving voorbereiden
3. Reconnaissance
4. Scanning en enumeration
5. Exploitatie (alleen op test-omgeving)
6. Post-exploitatie analyse
7. Rapport met bewijs en mitigatie
8. J.-review en goedkeuring

### 3. Incident Response

1. Detectie en initial assessment
2. Containment (scope beperken)
3. Eradication (root cause verwijderen)
4. Recovery (systemen herstellen)
5. Lessons learned
6. Documentatie bijwerken

### 4. Compliance Audit

1. Framework selectie (ISO 27001 / NIST / GDPR)
2. Controls evalueren
3. Gap analyse
4. Remediation plan
5. Follow-up audit

## Quality Standards

- Alle tests geautoriseerd door J.
- Chain of custody bij forensics
- `[Bron: CVE/config]` bij feiten
- Geen productie-impact zonder expliciete goedkeuring
